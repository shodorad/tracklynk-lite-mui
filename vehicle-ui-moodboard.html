05
Shared Patterns
What every single one of them got right — that TrackLynk should too.
🌑
Dark-first, always
All 4 apps ship dark mode as default. Glare, premium feel, better contrast for data at a glance.
Tesla · Rivian · BMW · Mercedes
🎯
One accent, used sparingly
Red, green, cyan, gold — one per brand. Used only for interactive and alert states. Never decorative.
All four brands
🚗
Car as hero image
A photo or 3D render of the vehicle is the home screen centerpiece. Data wraps around the car, not the other way.
Tesla · Rivian · BMW · Mercedes
⚡
Persistent status chip
"Parked at 12:34" / "Charging 87%" is always visible. Never buried in a sub-screen.
Tesla · BMW · Mercedes
🗺
Map always present
Location is ambient — in the background or as a persistent mini-map, not a destination you navigate to.
Tesla · Rivian · BMW
🌊
Spring physics motion
No linear or ease-in-out anywhere. Every animation uses spring stiffness + damping. The UI feels physical.
Tesla · Rivian · Polestar
📊
Radial data displays
Circular progress rings for range, charge, efficiency. More spatial density than bars, more emotional too.
Rivian · Tesla · Polestar
🎯
TrackLynk's gap
The existing app uses teal (#00BFA5) — an orphaned color that matches none of these patterns and directly contradicts the brand coral.
Fix this first.
06
Apply to TrackLynk
What the reference demands. Specific, not vague.
Stop
What's holding it back
  • Using teal (#00BFA5) in the app — every competitor uses a single deliberate accent. This one doesn't match anything.
  • Map as a destination screen — Tesla, BMW, Mercedes all make location ambient, not a tap away.
  • Vehicle status buried in a sub-page — it needs to be persistent, first thing you see.
  • Generic form UI in onboarding — device setup should feel like you're connecting to your actual car.
  • Linear easing on transitions — the whole category runs on spring physics. Yours still feels mechanical.
Start
What would make it tier-1
  • Commit to coral (#E8656A / #C42B35) everywhere in the app, prototype, and marketing. Burn the teal.
  • Add a persistent vehicle card to home — last seen location, status chip, last trip summary. Always visible.
  • Use a vehicle silhouette on the device setup screen — show the OBD port location on a car illustration.
  • Spring stiffness 380 / damping 38 on screen transitions. Stiffness 300 / damping 25 on card entrances.
  • Map-first home screen — vehicle marker on the map, geofence zones drawn on the surface, history as route overlay.
WS
The whitespace no one is occupying

Tesla and Rivian are for one brand of premium hardware. TrackLynk is for every car built since 2008.

None of these competitors target multi-vehicle households, parents monitoring teenage drivers, or small business owners managing a few vans. That practical, any-car-any-person positioning is a wide-open lane — but the UI has to signal "serious tool" not "budget tracker." It can be polished and operational simultaneously. That's the design brief.
