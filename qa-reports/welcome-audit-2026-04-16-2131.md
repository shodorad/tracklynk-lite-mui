**Critical finding:** There is NO centralized token file. Every color, spacing, radius, and font weight is hardcoded as inline style values scattered across 8+ screen components. The brand color `#E8656A` appears as a raw hex literal dozens of times. This is the single biggest systemic issue in this codebase.

## Findings

### 🔴 Critical

1. **Tagline text contrast — too low**
   - Visual: "Know where your vehicle is — always." is barely readable against the dark background
   - DOM: `color: rgba(255,255,255,0.38)` — that's 38% white opacity
   - Token: No token defined; hardcoded inline
   - Diagnosis: **Token gap + accessibility failure.** WCAG AA requires 4.5:1 contrast for normal text. rgba(255,255,255,0.38) on `#060810` yields roughly **2.8:1** — fails AA. This same `0.38` opacity is used for subtitles across ALL three slides ("No contracts. Cancel any time.", "One device. Full vehicle intelligence.") making it a systemic violation.

2. **No design token system exists**
   - Visual: Looks consistent by coincidence — the same developer wrote all screens
   - DOM: Every value is a hardcoded inline style literal
   - Token: N/A — no token file exists at all
   - Diagnosis: **Both broken.** The brand color `#E8656A` appears as a raw string 20+ times. Spacing values (8, 10, 14, 16, 18, 24, 44) are scattered with no scale. Border-radius values (7, 9, 12, 13, 16, 18, 20, 48, 99) have no pattern. One developer changing a color means find-and-replace across 8 files. This is a maintenance time bomb.

3. **Carousel dot indicators are not keyboard-accessible or labeled**
   - Visual: Three small dots at bottom of carousel — purely visual divs
   - DOM: `<div>` elements with `onClick` handlers, no `role="tab"`, no `aria-label`, no `tabIndex`
   - Token: N/A
   - Diagnosis: **Accessibility violation.** Screen readers can't navigate or identify these controls. Keyboard users can't tab to them.

### 🟡 Warnings

4. **Pricing slide — excessive empty space above "Simple pricing" heading**
   - Visual: ~120px of dead space between status bar and heading on Slide 3
   - DOM: The carousel area uses `justifyContent: 'center'` but the pricing content block is shorter than slides 1 and 2, so it floats with unbalanced whitespace above
   - Token: Padding top is `0 24px` in the slide, but parent has `paddingTop: 44` for status bar
   - Diagnosis: **Rendering issue.** The vertical centering works differently depending on content height. Slide 1 (hero with car image) fills nicely. Slide 3 leaves a visible gap at top.

5. **"Test drive the app" button — low text contrast**
   - Visual: Text is readable but muted
   - DOM: `color: rgba(255,255,255,0.72)` on `background: rgba(255,255,255,0.06)` over `#060810`
   - Token: No token; hardcoded inline
   - Diagnosis: **Token gap.** The effective contrast is ~5.2:1 which passes AA, but the 0.72 opacity makes it feel intentionally deemphasized. Fine as a design choice, but borderline for a secondary CTA that you actually want people to tap.

6. **Trust badge icons too small to be meaningful**
   - Visual: Three tiny car icons (13x9px SVGs) next to "10,000+ vehicles protected" — they look like red blobs at actual size
   - DOM: SVG paths are correctly rendered but at insufficient size
   - Token: `width: 24, height: 24` container, `width: 13, height: 9` SVG content
   - Diagnosis: **Visual issue.** The icons serve no communicative purpose at this size. Either enlarge them or replace with a simpler social-proof pattern (avatars, stars, etc.).

7. **Inconsistent border-radius scale**
   - Visual: Not immediately noticeable but creates subtle visual inconsistency
   - DOM: Values found: 7px (trust badge), 9px (toggle pill), 12px (toggle container), 13px (feature icon), 16px (secondary button), 18px (primary button, feature cards), 20px (price card), 48px (phone frame), 99px (dots, checkmarks)
   - Token: No radius scale defined
   - Diagnosis: **Token gap.** 9 distinct radius values with no governing scale. Some are close enough to look accidental (7 vs 9, 16 vs 18 vs 20).

8. **"Sign In" text button — tap target too small**
   - Visual: "Already have an account? Sign In" is a full-width button but only has `padding: 4px 0`
   - DOM: Computed height is roughly 22px content + 8px padding = ~30px
   - Token: Hardcoded `padding: '4px 0'`
   - Diagnosis: **Accessibility warning.** Apple HIG recommends 44px minimum touch target. This is well under that threshold.

### 🔵 Notes

9. **Phone frame looks polished — the box-shadow layering is well-crafted**
   - The multi-layer shadow (`0 0 0 1px`, `0 0 0 10px`, `0 48px 120px`, `0 0 80px`) creates a convincing device bezel. Nice work.

10. **Carousel auto-advance at 4.5s is reasonable but there's no pause-on-hover**
    - If a user is reading the pricing slide and it auto-advances, they'll have to navigate back. Consider pausing auto-advance when the user has interacted (dragged/tapped).

11. **Inter font is loaded correctly and renders well**
    - All text uses `Inter, sans-serif` consistently across all slides. Weight range (400-900) is appropriate. The tight letter-spacing on headings (-1.8px, -0.8px) gives a modern feel.

12. **Glass morphism effect is subtle and tasteful**
    - `rgba(255,255,255,0.055)` with `blur(20px) saturate(160%)` — not overdone. The borders at `rgba(255,255,255,0.10)` add just enough definition.

## Recommendations

**Priority 1 — Fix now:**
1. **Create a token file** (`src/theme/tokens.js`). Extract the brand color `#E8656A`, background colors, opacity scale, spacing scale, and radius scale into one source of truth. This single change prevents every other issue from recurring. Without it, you're playing whack-a-mole.
2. **Raise subtitle text opacity** from 0.38 to at least 0.55 to pass WCAG AA. Test with a contrast checker against `#060810`.
3. **Add `role="tab"`, `aria-label`, and `tabIndex={0}`** to the carousel dot indicators.

**Priority 2 — Fix soon:**
4. **Normalize border-radius** to 3-4 values (e.g., `sm: 8`, `md: 14`, `lg: 20`, `full: 99`). Map every component to one of these.
5. **Increase "Sign In" button tap target** to at least 44px height.
6. **Fix vertical spacing on Pricing slide** — either add a top element or adjust justifyContent to `flex-start` with explicit top padding matching other slides.

**Priority 3 — Consider:**
7. Replace trust badge car icons with something recognizable at small sizes (checkmarks, user avatars, or just the number with a shield icon).
8. Add `pause on interaction` logic to the carousel auto-advance timer.
9. Build the spacing scale into Tailwind config rather than using inline styles — you're already paying the Tailwind tax but getting none of the benefits.
